# レシピ管理アプリ 要件定義書

## 1. 概要

Google Keep のデザイン・インターフェースを参考にした、レシピ管理 Web アプリケーション。
Next.js App Router をフロントエンド兼 BFF とし、Supabase（PostgreSQL + Storage）を BaaS として利用する。

| 項目 | 内容 |
|------|------|
| アプリ名（仮） | Recipes |
| 表示言語 | 日本語固定 |
| デプロイ先 | Vercel（本番のみ、ステージングなし） |
| 想定規模 | 利用者 〜10人、レシピ 〜500件 |

---

## 2. 技術スタック

| レイヤ | 技術 |
|--------|------|
| フレームワーク | Next.js 16（App Router） |
| UI | React 19、Tailwind CSS 4 |
| BaaS | Supabase（PostgreSQL、Storage） |
| 認証 | カスタム（ユーザーテーブル + セッション） |
| メール送信 | Resend |
| ホスティング | Vercel |

---

## 3. ユーザー・認証

### 3.1 ロール

| ロール | 説明 |
|--------|------|
| `admin` | 管理者 |
| `user` | 一般ユーザー |

### 3.2 権限

| 操作 | admin | user |
|------|:-----:|:----:|
| ユーザー作成・削除 | ✅ | ❌ |
| 全レシピの CRUD | ✅ | ✅ |
| タグの自由入力・付与 | ✅ | ✅ |
| タグの編集・削除・統合 | ✅ | ❌ |

### 3.3 認証方式

- ログイン ID は**文字列**（メールアドレスはログインに使用しない）
- パスワードはユーザーテーブルで管理（ハッシュ化して保存）
- ユーザー登録は **admin のみ**が実行可能（自己登録不可）
- パスワードリセット・変更機能は**不要**（管理者が対応）
- セッション有効期限: 特に指定なし（一般的な設定、例: 7日間）

### 3.4 ユーザーテーブル項目

| カラム | 説明 |
|--------|------|
| id | ログイン用 ID（文字列） |
| name | 表示名 |
| role | `admin` / `user` |
| password | ハッシュ化されたパスワード |
| email | メールアドレス（admin への通知用。user も保持可） |

## 4. データモデル

### 4.1 レシピ

全ユーザーでレシピを**共有**する。全員が全レシピを閲覧・編集・削除できる。

| 項目 | 型 | 説明 |
|------|-----|------|
| タイトル | テキスト | 必須 |
| 材料 | テキスト | 自由記述（改行区切り） |
| 作り方 | テキスト | 自由記述（改行区切り） |
| その他 | テキスト | 自由メモ |
| 画像 | 複数 | Supabase Storage に保存 |
| タグ | 複数 | レシピに複数付与可能 |
| 色 | 色 ID | Google Keep 風のカード色分け |
| 作成日 | 日時 | 詳細画面に表示 |
| 更新日 | 日時 | 詳細画面に表示 |
| 削除日 | 日時 | 論理削除（ゴミ箱）用 |

### 4.2 画像

| 要件 | 内容 |
|------|------|
| 枚数上限 | 特に指定なし（実装側で合理的な上限、例: 10枚） |
| ファイルサイズ | 特に指定なし（例: 5MB/枚） |
| 対応形式 | JPEG、PNG、WebP |
| 一覧表示 | 複数枚を小さなサムネイルで並べて表示 |
| 並び順 | アップロード順（並び替え機能なし） |

### 4.3 タグ

| 要件 | 内容 |
|------|------|
| 作成 | レシピ登録時に自由入力（既存タグの候補表示あり） |
| 色 | Google Keep 風の色を付与 |
| 管理 | admin のみ編集・削除・統合可能 |
| サイドバー並び順 | 使用回数順（多い順） |
| 未選択時 | 全レシピを表示 |
| 複数選択時 | **AND** 条件 |

## 5. 機能要件

### 5.1 メイン画面

- Google Keep と同様の **Masonry Layout** でレシピカードを表示
- 左サイドバーにタグ一覧を表示
  - PC: 常時表示
  - モバイル: ハンバーガーメニューで開閉
- ライトモードのみ（ダークモード非対応）
- レスポンシブ対応必須

### 5.2 レシピ CRUD

- **新規作成・編集**: メイン画面からインラインで展開（Keep 同様）
- **詳細表示**: モーダル（オーバーレイ）
- **削除**: 論理削除（ゴミ箱へ移動、Keep 風）
- **複製**: 不要
- **エクスポート/インポート**: 不要
- **印刷用レイアウト**: 不要

### 5.3 検索

| 項目 | 内容 |
|------|------|
| 方式 | 形態素解析を含む高度な全文検索（日本語対応） |
| 対象 | タイトル、材料、作り方、その他 |
| 対象外 | タグ名 |
| タグ絞り込みとの組み合わせ | 検索結果 ∩ 選択タグ（AND） |
| 並び順 | タイトル順 |

### 5.4 ゴミ箱

- 削除されたレシピはゴミ箱に移動
- ゴミ箱からの復元・完全削除が可能（Keep 風）

### 5.5 メール通知

| 項目 | 内容 |
|------|------|
| トリガー | レシピの追加・変更・削除 |
| 送信先 | `admin` ロールの全ユーザー |
| メールアドレス | ユーザーテーブルの `email` カラム |
| 送信サービス | Resend |
| 通知内容（案） | 操作種別、レシピタイトル、操作者名、日時 |

---

## 6. 非機能要件

| 項目 | 内容 |
|------|------|
| アクセシビリティ | 基本的な対応（フォーカス管理、ラベル、セマンティック HTML） |
| エラー時 | エラーメッセージを表示 |
| オフライン時 | 読み取り不可の通知を表示 |
| パフォーマンス | 小規模運用（〜10人、〜500レシピ）を前提 |

---

## 7. インフラ・環境

| 項目 | 内容 |
|------|------|
| Supabase | 新規プロジェクトを作成 |
| ローカル開発 | リモート Supabase に直接接続（Supabase CLI ローカルは使わない） |
| デプロイ | Vercel（本番のみ） |
| ステージング | 不要 |

## 8. 画面一覧（想定）

|------|------|
| ログイン | ID + パスワード |

| レシピ詳細 | モーダル |
| ゴミ箱 | 削除済みレシピ一覧 |
| ユーザー管理 | admin のみ。ユーザー CRUD |
| タグ管理 | admin のみ。タグ編集・削除・統合 |
